## アーキテクチャ

```
packages/
  schema/          @world-forge/schema         — spatial types, validation, 2.5D fields
  export-ai-rpg/   @world-forge/export-ai-rpg  — AI RPG Engine export pipeline + CLI
  export-unreal/   @world-forge/export-unreal  — Unreal Engine 5 export pipeline + CLI (2.5D aware)
  export-godot/    @world-forge/export-godot   — Godot 4 export pipeline + .tscn scene generation
  renderer-2d/     @world-forge/renderer-2d    — PixiJS 2D canvas renderer
  editor/          @world-forge/editor         — React web authoring app
```

## クイックスタートガイド

```bash
npm install
npm run build
npm run dev --workspace=packages/editor
```

エディターを起動するには、`http://localhost:5173` を開いてください。

### 編集ワークフロー

1. **モードを選択** — ダンジョン、地区、ワールド、海、宇宙、屋内、または荒野の中からいずれかを選び、グリッドのデフォルト設定と接続語彙を設定します。
2. **キットから開始** — テンプレートマネージャーからスターターキットまたはジャンルテンプレートを選択するか、白紙の状態から始めます。
3. **ゾーンを描画** — キャンバス上でドラッグしてゾーンを作成し、それらを接続して地区を割り当てます。
4. **エンティティを配置** — NPC、敵、商人、イベント、アイテムなどをゾーンに配置します。
5. **確認** — 「レビュー」タブを開き、状態、コンテンツの概要、および要約エクスポート（Markdown/JSON）を確認します。
6. **エクスポート** — エクスポートモーダルを開き、ターゲットごとの準備状況（✓ 準備完了 / ⚠ 注意事項）、ターゲットオプションを設定し、AI RPG Engine、UE5、またはGodot 4のパッケージをダウンロードします。エクスポート後のレシートには、サイズ、数、詳細などが記録されます。また、プロジェクトバンドル（.wfproject.json）とレビュー要約も含まれます。

### コマンドラインインターフェースによるエクスポート

```bash
# AI RPG Engine
npx world-forge-export project.json --out ./my-pack
npx world-forge-export project.json --validate-only

# Unreal Engine 5
npx world-forge-export-unreal project.json --out ./UnrealPack --sign
npx world-forge-export-unreal --summary ./UnrealPack
```

## パッケージ

### @world-forge/スキーマ

ワールド制作に使用する、基本的なTypeScriptの型と検証機能。

- **空間タイプ** - `WorldMap`、`Zone`、`ZoneConnection`、`District`、`Landmark`、`SpawnPoint`、`EncounterAnchor`、`FactionPresence`、`PressureHotspot`
- **コンテンツタイプ** - `EntityPlacement`、`ItemPlacement`、`DialogueDefinition`、`PlayerTemplate`、`BuildCatalogDefinition`、`ProgressionTreeDefinition`
- **視覚レイヤー** - `AssetEntry`、`AssetPack`、`Tileset`、`TileLayer`、`PropDefinition`、`PropPlacement`、`AmbientLayer`
- **町と構造物** - `MarketNode`、`CraftingStation`、`Building`、`Hub`、`Stronghold`
- **ワールドモデリング** - `Stratum` + `StratumLink`（垂直レイヤー）、`HazardDefinition`（タイプ化されたエフェクトの集合）、`ZoneEntryGate` + パーティの状態を示す `SpawnCondition` のオペランド（`party-level`、`party-size`、`item`、`flag`、`member`、`class`）
- **モードシステム** - `AuthoringMode`（7つのモード）、モード固有のグリッド/接続/検証プロファイル
- **検証** - `validateProject()`（マップベースのO(n)検索による78個の構造チェック、`warningCount`）、`advisoryValidation()`（モード固有の提案、メタデータの完全性、アセットの名前付け）
- **ユーティリティ** - `assembleSceneData()`（欠落したアセットを検出する視覚的なバインディング）、`scanDependencies()`（参照グラフ分析）、`buildReviewSnapshot()`（健全性の分類）

### @world-forge/export-unreal

「WorldProject」を、2.5Dゲーム向けに最適化されたUnreal Engine 5のコンテンツパックに変換します。

- **出力**：`pack.json`、ゾーンごとおよび地区ごとの主要データアセットのJSON形式ファイル、グループ化されたアクター生成マニフェスト、接続ごとのレベルストリーミングヒント、ワールドパーティションセルのヒント、および構造化された品質レポート。
- **2.5Dフィールド**：`Zone.elevation`、`elevationRange`、`parallaxLayers`、`skylineRef`が保持され、UEのcm/Z軸方向座標に変換されます。
- **座標変換**：純粋な関数（`pixelsToUnrealCm`、`elevationToZ`、`worldForgeToUnrealAxis`、`gridToUnrealAxis`）。デフォルトの世界スケールは、1タイル＝100cmです。
- **双方向インポート**：`importFromUnreal`は、Unrealのパッケージからワールドプロジェクトを再構築します。ゲームプレイのみのデータ（会話、進行状況、ビルド）は、品質レポートで破棄されたものとしてフラグが立てられます。
- **CLI**：`world-forge-export-unreal`に、`--out`、`--tile-size-cm`、`--validate-only`、`--verbose`のオプションを指定します。

### @world-forge/export-godot

「WorldProject」を、`.tscn`形式のシーンテキストを含むGodot 4のコンテンツパックに変換します。

- **出力** — `pack.json`、ゾーンごとのリソース、エンティティマニフェスト、ナビゲーションリンク、略奪テーブル、スポーンマーカー、トランジションノード、ダイアログリソース、アセットバインディング、およびワールドの`.tscn`シーン
- **プレイ可能なシーン** — `buildWorldScene()`は、移動可能な`.tscn`を出力します。これには、ゾーンごとの`StaticBody2D`コリジョンと`NavigationRegion2D`、フレーム化された`Camera2D`、およびY軸ソート/`z_index`深度が含まれます。
- **タイルと内部** — `TileMapLayer` + `TileSet`（画像タイルセット用にベイクされた`tile_map_data`）、セルごとの壁の`StaticBody2D`コリジョン、およびプロップの`Node2D`配置
- **町** — 市場と作成ステーション、および建物（`StaticBody2D`フットプリント）/ハブ/拠点。これらはすべて、メタデータとしてデータを保持する`Node2D`プレースホルダーとして存在します。
- **ワールドモデリング** — 垂直層（ゾーンごとの`z_index`バンドと`StratumLink`コネクタ）、タイプ分けされた危険要素を`Area2D`領域として、およびゾーンエントリーゲートのメタデータ
- **忠実度レポート** — 無損失、近似、および破棄されたデータの構造化された追跡。これは、実際のGodot 4エンジンに対して検証されます（ヘッドレススモーク、36個のアサート）。
- **フォーマットバージョン** — `GODOT_PACK_FORMAT_VERSION` 1.0.0

### @world-forge/export-ai-rpg

「WorldProject」を、ai-rpg-engineで使用される「ContentPack」形式に変換します。

- **エクスポート**：ゾーン、地区、エンティティ、アイテム、ダイアログ、プレイヤーテンプレート、ビルドカタログ、プログレッションツリー、遭遇イベント、派閥、ホットスポット、マニフェスト、およびパックメタデータ。
- **インポート**：8つの逆変換ツールが、エクスポートされたJSONからWorldProjectを再構築します。
- **忠実度レポート**：変換中にロスなく保持されたもの、近似的に処理されたもの、または削除されたものを構造化して追跡します。
- **フォーマット検出**：WorldProject、ExportResult、ContentPack、およびProjectBundleのフォーマットを自動で検出し、識別します。
- **CLI**：`world-forge-export`コマンドに、`--out`、`--validate-only`、および`--verbose`フラグを追加して使用します。

### @world-forge/レンダラー2D

PixiJSをベースにした2Dレンダラー：パン／ズーム機能付きのビューポート、地域ごとに色分けされたゾーンオーバーレイ、接続を示す矢印、役割に応じたエンティティアイコン、タイルレイヤー、およびミニマップ。

### @world-forge/エディター

React 19とViteを使用したウェブアプリケーション。Zustandによる状態管理、アクションラベル付きの取り消し/やり直し機能、自動保存（30秒間隔、3バージョンの履歴、クラッシュからの復旧）、プロジェクト読み込み時のすべてのパスにおけるダーティステートガード、ダークモード/ライトモードの切り替え、モーダルウィンドウのフォーカス制御、キーボード操作によるツール切り替え機能を実装。

#### キャンバスと編集

- **ツール**：選択、領域塗りつぶし、接続、エンティティ配置、ランドマーク、スポーン
- **複数選択**：シフトクリック、ボックス選択、Ctrl+A；アトミックな元に戻し機能付きドラッグ移動
- **整列**：6方向の整列（左/右/上/下/水平中央/垂直中央）と水平/垂直方向への配置
- **スナップ**：視覚的なガイドラインを表示しながら、近くのオブジェクトのエッジや中心にドラッグ時に自動的に吸着させる機能
- **サイズ変更**：各ゾーンに8つのハンドルを設け、エッジのスナップ、最小サイズの制限、リアルタイムプレビュー機能を搭載
- **複製**：Ctrl+DでID、接続、およびエリアの割り当てを変更して複製
- **コピー/ペースト**：Ctrl+C / Ctrl+VでIDを変更し、オフセットを設定してコピー/ペースト
- **クリックサイクル**：同じ位置を繰り返しクリックすると、重なり合ったオブジェクトが順番に選択される
- **コンテキストメニュー**：右クリックで7つの状況に応じたアクション（プロパティ、削除、複製など）を表示
- **接続プレビュー**：接続ツールを使用中に、点線状のシアン色のラインを表示して接続先を示す
- **ミニマップ**：200×150ピクセルの概要図（右下）、クリックするとその位置に移動する
- **ビューポートカリング**：表示範囲内のオブジェクトのみをレンダリングする（64ピクセルのマージン）
- **パフォーマンス統計**：FPS、オブジェクト数、レンダリング時間をオーバーレイで表示/非表示を切り替え可能
- **個別のオブジェクトの表示/非表示**：個々のオブジェクトを表示または非表示にする（localStorageに設定を保存）
- **レイヤー**：7つの表示/非表示切り替え（グリッド、接続、エンティティ、ランドマーク、スポーン、背景、環境光）

#### ナビゲーションとショートカットキー

- **ビューポート** — カメラのパン/ズーム、マウスホイールによるズーム（カーソルを基準）、スペースキー／中央クリック／右クリックによるドラッグ操作での画面移動、コンテンツに自動的に合わせて表示、ダブルクリックで中心に表示
- **検索** — Ctrl+K でオーバーレイを開き、あいまい一致により名前またはIDで任意のオブジェクトを検索。キーボード操作によるナビゲーションと最近の検索履歴（ローカルストレージ）に対応。
- **スピードパネル** — ダブル右クリックすると、コンテキストに応じたアクション、お気に入り、マクロ、モードに合わせたクイックアクションを備えたフローティングコマンドパレットが表示されます。
- **ホットキー** — ツール切り替え（V/Z/C/E/L/S）、詳細表示（Enter）、プリセットの適用（P）、プリセットの保存（Shift+P）、コピー／ペースト（Ctrl+C/V）、矢印キーによる微調整（Shift = 5倍）など、21個のキーボードショートカット。
- **アクセシビリティ** — Escapeキーで閉じるモーダルフォーカストラップ、すべてのアイコンのみのボタンにARIAラベルを付与、キーボード操作可能なオブジェクトツリー、スクリーンリーダーによる変更インジケーターのアナウンス。空間キャンバス操作（配置、ボックス選択、サイズ変更、接続線の描画、画面移動）は引き続きポインタベースで行われます。

#### 輸出入

- **コンテンツパック**：AI RPGエンジン、Unreal Engine 5、またはGodot 4へのターゲットに合わせたエクスポート。各ターゲットに対応するステータスバッジ、設定可能なオプション（タイルサイズ、シーンのプレフィックス、バンドルフィルタリング）、およびダウンロード後の領収書が含まれます。
- **プロジェクトバンドル**：Provenanceメタデータと依存関係情報を含む、移植可能な`.wfproject.json`ファイル。
- **キットバンドル**：検証、コリジョン処理、Provenance追跡機能付きの`.wfkit.json`のエクスポート/インポート。
- **インポート**：構造化された忠実度レポートとともに、4つの形式を自動的に検出します。
- **差分比較**：インポート以降のセマンティックな変更を追跡します。
- **シーンプレビュー**：すべてのゾーンの視覚的なバインディングをインラインHTML/CSSで構成し、表示します。

## コンテンツ作成方法

ワールドフォージでは、**ジャンル**（ファンタジー、サイバーパンク、海賊）と**モード**（ダンジョン、海洋、宇宙）を区別します。ジャンルはゲームの雰囲気を決定し、モードは規模を決定します。モードによって、グリッドのデフォルト設定、接続に使用する語彙、検証に関する提案、ガイドの説明文、およびプリセットフィルターが制御されます。

| モード | グリッド | タイル | 重要なつながり |
|------|------|------|-----------------|
| ダンジョン | 30×25 | 32 | ドア、階段、通路、秘密の場所、危険な場所 |
| 地区／都市 | 50×40 | 32 | 道、ドア、通路、入口 |
| 地域／世界 | 80×60 | 48 | 道、入り口、通路 |
| 海 | 60×50 | 48 | 経路、ルート、ポータル、危険物 |
| 空間、宇宙 | 100×80 | 64 | ドッキング、ワープ、通路、ポータル |
| 内装 | 20×15 | 24 | ドア、階段、通路、秘密の場所 |
| 荒野、自然のままの場所 | 60×50 | 48 | 小道、道路、通路、危険箇所 |

プロジェクトを作成する際にモードを設定し、その設定は「ワールドプロジェクト」の`mode?: AuthoringMode`として保存されます。各モードには、接続の種類、エンティティの役割、ゾーン名、およびスピードパネルの提案など、**便利なデフォルト設定**が用意されており、これらは自動的に調整されます。

## コンテンツ作成画面

### 世界の構造

- 空間的なレイアウト、隣接するエリア、出口、光、騒音、危険箇所、およびインタラクション可能なオブジェクトを含むゾーン
- 12種類の接続タイプ（通路、ドア、階段、道、ポータル、秘密の場所、危険箇所、チャネル、ルート、ドッキングステーション、ワープ、トレイル）。それぞれ異なる視覚スタイルを持ち、エッジに固定されたルーティング、方向を示す矢印、条件付きの破線スタイルが適用される
- 勢力による支配、経済プロファイル、メトリクススライダー、タグ、およびゾーンの中心部に配置された地区名ラベルを持つ地区
- ランドマーク（ゾーン内の名前付きの興味深い場所）
- スポーンポイント、遭遇地点（タイプに基づいた色の指定）、勢力の存在、および危険なホットスポット
- **垂直層** - 個別のレイヤー（表面/地下/空、または建物の階）。各レイヤーには順序、Z範囲、レイヤー間の可視性、およびコネクター（階段/はしご/エレベーター）が設定される。ゾーンは特定の層に割り当てられる
- **タイプ指定された環境の危険要素** - 共有の危険要素ライブラリ（ダメージ/ステータス/即死/発火効果、トリガータイミング、地形の移動コスト、通行可能性、視界の遮断、天候による制限）。各ゾーンで参照される
- **ゾーンへの進入ゲート** - パーティーの状態（レベル/人数/アイテム/フラグ/メンバー/クラス）に基づいてゲートが開き、ハードまたはアドバイザリーゲートとして機能する。ゲートには、「ロックを表示」するための理由が設定されている

### コンテンツ

- ステータス、リソース、AIプロファイル、およびカスタムメタデータを持つエンティティの配置
- スロット、レア度、ステータスマジファイア、および付与されるアクションを持つアイテムの配置
- 分岐する会話、条件、および効果を含むダイアログツリー
- キャンバス上の遭遇地点（赤い菱形のマーカー。ボス/待ち伏せ/パトロールタイプを示す）

### 街と内部空間

- タイルペイント - 画像を背景としたタイルセット（行/列でスライス）。色のついた長方形のフォールバック、ドラッグブラシ、レイヤー、および壁との衝突判定のための各タイルの「Solid」歩行可能性を持つ
- 内部空間用のプロップ配置（パレット+キャンバスレンダリング）、配置ツール付き
- 街の経済 - 市場ノード（供給カテゴリ、価格修正係数、違法品）、およびクラフトステーション（ステーションタイプ、レシピ）。各ゾーンで編集可能
- 街の構造物 - 建物（内部ゾーンへのリンクを持つ、立ち入ることができるフットプリント）、ハブ（サービス+接続ノード）、および要塞（強化された勢力の拠点）

### キャラクターシステム

- プレイヤーテンプレート（初期ステータス、インベントリ、装備、スポーンポイント）
- ビルドカタログ（アーキタイプ、背景、特性、専門分野、クロスタイトル、関係性）
- プログレッションツリー（要件と効果を持つスキル/アビリティノード）

### 資産

- アセットマニフェスト（ポートレート、スプライト、背景、アイコン、タイルセット）。種類ごとに特定のバインディングを持つ
- アセットパック（名前付き、バージョン管理されたグループ。互換性メタデータ、テーマ、ライセンスを含む）
- シーンプレビュー（ゾーンのすべての視覚的バインディングをインラインで構成し、アセットが不足している場合に検出する）

### ワークフロー

- リージョンプリセット（9種類の内蔵プリセット、モードによるフィルタリング）、および遭遇プリセット（10種類の内蔵プリセット）。マージ/上書きの適用とカスタムプリセットの作成/読み取り/更新が可能
- スターターキット（7種類の内蔵プリセット、モード固有）。キットのエクスポート/インポート（`.wfkit.json`）、衝突処理、および出所の追跡が可能
- レイアウトテンプレート（6種類の事前に構築されたゾーン配置）とダイアログテンプレート（5種類の会話の開始点）
- ゾーンのマージとバッチエンティティ配置（グリッド/ランダム/円形のパターン）
- 30秒間隔で自動保存し、3バージョンの復元履歴を保持する
- Ctrl+Kを押すと、あいまい一致と最近の履歴を使用して、すべてのオブジェクトタイプを検索できる
- ピン留め可能な「お気に入り」、マクロ、カスタムグループ、およびモードの提案を備えたスピードパネルコマンドパレット
- 21種類の集中管理されたキーボードショートカット（6つのツール切り替えキーを含む）
- プロジェクトメタデータエディター（作成者、ライセンス、カテゴリ、タグ）
- レビュー統計（役割の分布、接続タイプ、遭遇タイプ、地区ごとのゾーン数）
- ContentPack JSON、プロジェクトバンドル、およびレビューサマリーへのエクスポート
- 構造化された忠実度レポート、修正提案、およびセマンティック差分追跡を備えた4つの形式からのインポート

現在の状態を証明するチャペル・スレッショルドのエクスポートハンドシェイクについては、[`dogfood/WALKTHROUGH.md`](dogfood/WALKTHROUGH.md)を参照してください。

## ドッグフードディレクトリ

`dogfood/`ディレクトリには、ユニットテスト外で完全な作成からエクスポートまでのパイプラインを検証するための統合テストハーネスが含まれています。チャペル・スレッショルドの例（`chapel-threshold.ts`）は、小さくても完全なワールドプロジェクトを作成し、それをエクスポート処理にかけ、出力を`dogfood/output/`に書き込みます。これにより、スキーマタイプ、検証、およびエクスポートパイプラインが実際のデータでエンドツーエンドで機能することが証明されます（単なる孤立したモックではありません）。

## エンジンとの互換性

エクスポートは3つのエンジンを対象としています。

- **[ai-rpg-engine](https://github.com/mcp-tool-shop-org/ai-rpg-engine)** - ContentPack形式で、[claude-rpg](https://github.com/mcp-tool-shop-org/claude-rpg)にロード可能
- **Unreal Engine 5** - プライマリデータアセット、アクターのスポーンマニフェスト、およびワールドパーティションヒントを備えた2.5D対応コンテンツパック
- **Godot 4** - ゾーンリソース、ナビゲーションリンク、およびエンティティマニフェストを持つ`.tscn`シーン生成

## セキュリティ

- **アクセスされるデータ:** ローカルディスク上のプロジェクトファイル（ユーザーが作成したJSON）。サーバー側のストレージは使用しない
- **アクセスされないデータ:** テレメトリ、分析、ローカル開発サーバー以外のネットワークリクエストは行わない
- **権限:** APIキー、秘密鍵、認証情報は使用しない
- **ソースコードに秘密鍵、トークン、または認証情報は含まれない**
